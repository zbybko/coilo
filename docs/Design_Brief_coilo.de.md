# Design Brief — coilo.de (для Claude Design)

Бриф на обновление лендинга coilo.de. Все ассеты лежат в папке `site-assets/` рядом с этим файлом. Бренд — премиальный, минималистичный, «design object». Сайт остаётся одностраничным лендингом + страница `/colors` и `/about`.

Текущий стек: кастомный лендинг coilo.de → кнопки покупки ведут на `coilo.myshopify.com/cart/<variantId>:1` (Shopify checkout).

---

## 0. Сводка задач (чек-лист)

1. Заменить старые изображения новыми lifestyle-рендерами (webp в `site-assets/`).
2. Поднять цену **€39 → €59** во всех местах сайта.
3. Убрать/задвинуть Etsy-CTA, заменить главный посыл на «выбрать цвет».
4. Hero: цвет текста синхронизировать с цветом спирали при скролле (сейчас чёрный — заменить).
5. Color-конфигуратор: слайдер изображений выбранного цвета.
6. Перенести отзывы с Etsy (тексты ниже).
7. Добавить email `support@coilo.de`.
8. Добавить картинку размеров (`shared/size-infographic.webp`).
9. Добавить видео: рекламный ролик (`video/ad-video.mp4`) + таймлапс печати (Zakhar загрузит отдельно).

---

## 1. Цена: €39 → €59

Заменить **везде**: блок характеристик («PRICE €39» → «€59»), карточки цветов в конфигураторе («€39» → «€59»), любые мета/structured data.

ВАЖНО: цена в Shopify-чекауте берётся из варианта товара, не из лендинга. Цену вариантов на Shopify меняет Zakhar отдельно (Cowork помогает) — иначе на лендинге будет €59, а в корзине €39. Этот пункт отметить как зависимость, не правится в коде лендинга.

Опционально: показывать перечёркнутую «launch»-цену, если будет промо.

## 2. Изображения (новые ассеты)

Все оптимизированы под веб (webp, 100–230 КБ). Структура:

```
site-assets/
  hero/        hero-cherry.webp, hero-cyan.webp   (кандидаты на hero, 2400px)
  colors/
    cyan/      cyan-1.webp (реалистичный, дуб) · cyan-2.webp (океан) · cyan-3.webp (галерея) · cyan-studio.webp
    cherry/    cherry-1.webp · cherry-2.webp · cherry-studio.webp · cherry-studio-2.webp
    sakura/    sakura-1.webp · sakura-studio.webp · sakura-studio-2.webp
    sunflower/ sunflower-1.webp · sunflower-studio.webp
    rose/      rose-1.webp · rose-studio.webp
  shared/      size-infographic.webp · logo.png
  video/       ad-video.mp4
```

Замены:
- Старый `Cyan.png` в hero/секциях → новые рендеры.
- `-1/-2/-3` = lifestyle (интерьер), `-studio` = товар на чистом фоне.
- Везде задать `alt` (напр. «Coilo Spiral Bookshelf in Cyan on oak sideboard»), `loading="lazy"` для несекций-hero, `width`/`height` против CLS.

Примечание о контенте: у Cyan/Cherry есть по 2–3 lifestyle-кадра, у Sakura/Sunflower/Rosé — по одному (дополнены студийными). Если для слайдеров нужно больше lifestyle на цвет — Cowork догенерит, скажи.

## 3. Hero: цвет текста, синхронизированный со спиралью

Сейчас: hero-рендер спирали меняет цвет при скролле, но текст статично **чёрный** — выглядит слабо.

Нужно: цвет hero-заголовков/подписей плавно меняется в такт текущему цвету спирали по мере скролла. Стартовые значения (Claude Design подгоняет под контраст/WCAG AA против фона):

| Финиш | Accent (яркий: имя цвета, акценты) | Text (глубокий тон для заголовков) |
|---|---|---|
| Cyan | `#1BA6DF` | `#0A4A63` |
| Sakura | `#F2A3BE` | `#8A3A52` |
| Cherry | `#C0303A` | `#5E1418` |
| Sunflower | `#F2A900` | `#6E4E00` |
| Rosé | `#F0457A` | `#7A1E3C` |

Поведение: интерполировать цвет текста между соседними финишами синхронно с цветом рендера (CSS-переменная `--spiral-color`, обновляется на scroll/в той же таймлайн-логике, что и рендер). Переход плавный (`transition: color .4s ease`). На очень светлых кадрах добавить лёгкую тень/подложку для читабельности.

## 4. Color-конфигуратор: слайдер выбранного цвета

Сейчас: выбор цвета меняет название/описание и одну картинку.

Нужно: при выборе цвета показывать **слайдер/карусель** изображений именно этого финиша (из `colors/<color>/`). Порядок: сначала lifestyle (`-1`, `-2`, `-3`), затем студийные (`-studio*`).

- Управление: стрелки + свайп на тач + точки-индикаторы.
- При смене цвета слайдер сбрасывается на первый кадр, акцент-цвет UI берётся из таблицы выше.
- Кнопка «Buy Now» под слайдером ведёт на нужный Shopify-вариант (маппинг variantId):

| Цвет | Shopify variantId | Cart-ссылка |
|---|---|---|
| Cyan | 61987185787210 | `https://coilo.myshopify.com/cart/61987185787210:1` |
| Sakura | 62010088554826 | `…/cart/62010088554826:1` (сейчас распродан — показать «Sold out») |
| Cherry | 62010088587594 | `…/cart/62010088587594:1` |
| Sunflower | 62010088620362 | `…/cart/62010088620362:1` |
| Rosé | 62010091077962 | `…/cart/62010091077962:1` |

## 5. Меньше Etsy, больше «выбрать цвет»

Сейчас сайт уводит трафик на Etsy (в меню «Shop on Etsy», секция «Visit our Etsy Shop», «Also available on Etsy»). Для продаж на своём сайте это контрпродуктивно.

- Навигация: «Shop on Etsy» → **«Shop»** или **«Pick a Color»**, скроллит к конфигуратору (не на Etsy).
- Главные CTA («Buy Now», «Shop Now», «Pick a Color») остаются и ведут в конфигуратор/Shopify-чекаут.
- Секцию «Also available on Etsy / Visit our Etsy Shop» — убрать как отдельный CTA. Вместо неё оставить **одну ненавязчивую строку доверия** внизу: «Trusted by buyers on Etsy · ★★★★★» (маленькая ссылка, не кнопка). Etsy используем как соцдоказательство, а не как канал ухода.

## 9. Видео

- **Рекламный ролик** — `video/ad-video.mp4` (готов). Разместить в hero как фон/вставку ИЛИ отдельной секцией «See it in motion». Атрибуты для web-фона: `muted autoplay loop playsinline preload="metadata"`, постер-кадр (можно вытащить из видео), `loading="lazy"`.
- **Таймлапс печати** — Zakhar загрузит отдельным файлом. Идеальное место — страница/секция **«How It's Made»** (подкрепляет «Printed, not manufactured»). Зарезервировать контейнер `video/print-timelapse.mp4`.

Видео > ~6 МБ — дать сжатый MP4 (H.264) + постер; не автоплеить тяжёлое на мобильном.

## Как давать референсы на «всё остальное» (для следующих итераций)

Чтобы Claude Design попадал в точку без переделок, давай референсы по схеме «адрес → что → пример»:

1. **Привязывай к месту.** Скриншот нужной секции coilo.de + стрелка/подпись «вот тут». Claude Design правит код — ему важно, какой это блок.
2. **Показывай, а не описывай.** На каждую правку — 1–2 визуальных примера: скрин чужого сайта/Dribbble/Pinterest с пометкой «хочу такой слайдер / такую анимацию текста». Одна картинка экономит абзац текста.
3. **Складывай в одну папку.** Заведи `site-assets/refs/` с подпапками по задаче (`refs/hero-text/`, `refs/slider/`, `refs/reviews-section/`) — туда скрины-референсы. Claude Design берёт оттуда.
4. **Указывай поведение словами-триггерами.** Для интерактива опиши состояние: «при наведении…», «при скролле вниз…», «на мобильном…». Это снимает 90% догадок.
5. **Фиксируй, что НЕ трогать.** «Шрифты и сетку оставить как есть» бережёт то, что уже нравится.
6. **Бренд-константы — один раз.** Палитра (таблица §3), шрифты, тон голоса («design object», «where design unwinds») — собери в один `BRAND.md`, ссылайся на него, не повторяй в каждом брифе.

Шаблон одной правки:
```
ГДЕ: секция Hero (скрин refs/hero-text/current.png)
ЧТО: цвет заголовков синхронизировать со спиралью
КАК: см. палитра §3, переход плавный
РЕФЕРЕНС: refs/hero-text/example.png
НЕ ТРОГАТЬ: текст копирайта и шрифт
```
